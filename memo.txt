
버전관리도구 -> 소프트웨어 제품의 버전을 관리해주는 프로그램
-> git, svn, cvs ...

git : 작성된 코드(파일)의 이력을 관리해주는 프로그램

    주요개념
    local-repository : 개발자 컴퓨터에서 git으로 코드를 관리하는 저장소
    remote-repository : 원격 저장소 -> 네트워크상에서 제공하는 저장소
        -> 서버(github, gitlab, gittree ...)
    branch : 작업공간(코드를 수정, 보수)

    파일관리상태 (git)
        tracked : git에 의해 파일이 관리되고 있는 상태 -> index 부여된 상태
        untracked : git에 의해 파일이 관리되지 않는 상태 -> .gitignore에 등록된 파일

    관리되는 파일의 처리상태
        commited : 새로운 이력을 추가한 상태 -> 저장 -> commit 명령을 실행함
        modified : 파일을 수정한 상태 -> 수정해서 파일내용이 변경된 상태
        staged : 수정된 파일을 commit 하기 전 상태

git 으로 프로젝트 관리하기
    1. 프로젝트 생성
    2. local-repository 생성 -> git -> git init -> .git
        - 자동으로 기본 브랜치(master -> main) 생성 -> 프로젝트를 관리
    3. 프로젝트의 파일 중 관리할 파일을 선택 / 설정 -> 인덱스 부여 -> git add 경로/파일명
        - 지정된 파일은 tracked 상태로 변경
        - .gitignore 파일에 관리하지 않을 파일을 설정
    4. 프로젝트 개발 -> 코드 작성 -> 파일 변경상태 확인 -> git status 
        - modified 상태 -> staged 
    5. 파일 commit 처리하기 -> 새로운 이력을 저장 -> git commit -m "" -a
        - 대상이 되는 파일 : staged에 있는 파일
        - commit 할 때는 반드시 메세지를 기록

    --remote-repository 관리
    6. 원격 레포지토리 생성(github)
        - github에 계정 생성
        - repository 생성 -> remote-repository = https : url / key : ssh
    7. 원격 레포지토리와 로컬 레포지토리 연결(등록)
        - 생성한 원격 레포지토리를 등록 -> git remote add remote명칭(origin) url
    8. pull : 원격 레포지토리에 있는 데이터를 가져와 병합하는 것 -> git pull 원격 레포지토리명 [브랜치명]
    9. fetch : 원격 레포지토리에 있는 데이터를 가져오는 것 > 병합 x, 별도 명령어로 병합처리 해야함 -> git fetch 원격 레포지토리명 [브랜치명]
    10. push : 원격 레포지토리에 로컬 레포지토리 데이터를 전송 -> 해당 브랜치에 merge -> git push 원격 레포지토리명 브랜치명
    11. clone : 원격 레포지토리에 있는 데이터를 가져옴 -> 로컬 레포지토리를 생성 -> git clone url/ssh
